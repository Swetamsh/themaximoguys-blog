You don't need to learn everything before migration day. You need the right things in the right order.

Here's the 4-week learning path that builds the foundation you actually need as a MAS app developer:

1. Week 1: GraphQL Fundamentals. GraphQL is the primary data access layer in MAS 9. Learn query syntax, understand schemas and types, practice traversing related objects in a single query, and run your first queries against the MAS GraphQL endpoint. Four hours of focused learning and it clicks.

2. Week 2: REST API Patterns. REST is still how external systems trigger Maximo actions and integrations. Learn authentication, CRUD operations, error handling, and API testing with tools like Postman before you write a line of code.

3. Week 3: Container Basics. You don't need to become a Kubernetes operator. Learn what a container image is, what a pod does, and how your scripts land inside the MAS runtime. Read logs from a container. Understand why "works on my laptop" fails in MAS.

4. Week 4: Modern Dev Tooling. Git for everything — no more XML exports. VS Code with the GraphQL extension for schema-aware development. A basic CI/CD pipeline. Linting and code review basics. One week of tooling investment that pays dividends every day.

Four weeks. Practical. One week at a time. Your Maximo domain knowledge is already there — these four weeks give it a modern toolkit.

Save this for your team's migration prep.
