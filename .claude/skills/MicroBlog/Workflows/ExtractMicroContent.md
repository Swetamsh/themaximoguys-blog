# ExtractMicroContent Workflow

## Prerequisites

Before executing, load:
1. `VoiceGuide.md` — social voice adaptation rules
2. `ContentTypes.md` — micro-content type specifications

## Step 1: Read and Parse the Source Blog

Read the MDX file. Extract:
- **Frontmatter**: title, description, slug, tags, keyTakeaways, tldr, faqs, series, semanticKeywords
- **Body content**: section headers, blockquotes, comparison tables, bold statements, numbered lists

Record the source reference:
```yaml
sourceBlog:
  title: "[from frontmatter]"
  slug: "[from frontmatter]"
  series: "[from frontmatter series.name]"
  part: [from frontmatter series.part]
  url: "[relative path to source MDX file]"
```

## Step 2: Identify Atomic Thoughts

Scan the blog content for extraction signals. An **atomic thought** is a single insight that can stand alone without surrounding context.

### Extraction Signals

| Signal | Where to Look | Content Type |
|---|---|---|
| Blockquotes with bold text | `> **...**` | Hook |
| "No.", "Stop.", "Here's the truth" | Any sentence | Hook, Hot Take |
| Comparison tables (From/To) | Markdown tables with old vs new | Contrast, Carousel |
| `keyTakeaways` array | Frontmatter | Hot Take, Carousel slides |
| `faqs` array | Frontmatter | FAQ Post |
| Numbered headings (###) | Body sections | Thread tweets, Listicle items |
| Sentences with "NOT" / "not a...but a..." | Body | Hot Take |
| Rhetorical questions | Body | Hook |
| "Old world / MAS world" patterns | Body | Contrast |
| Numbered archetypes / categories | Body | Thread, Listicle |
| `tldr` | Frontmatter | Summary carousel slide |

## Step 3: Generate Micro-Content Pieces

For each atomic thought, write the micro-content following the rules in `VoiceGuide.md` and format specs in `ContentTypes.md`.

### Generation Order (by priority)

1. **Hooks** (2-3) — from blockquotes, bold opening lines, paradigm statements
2. **Thread** (1) — from the blog's main argument arc or numbered sections
3. **Carousel content** (1-2) — from comparison tables, key takeaways
4. **Hot Takes** (2-3) — from myth-busting sections, contrarian insights
5. **FAQ posts** — all FAQ entries from frontmatter
6. **Listicle** (1) — from numbered sections or key takeaways
7. **Contrast posts** (1-2) — from From/To tables

### Writing Rules

- **Stand alone**: Reader gets full value without the source blog
- **TMG voice**: Empathetic Expert, adapted per platform (see VoiceGuide.md)
- **No hedging**: Bold, direct, decisive
- **One idea per piece**: If you're explaining two things, split into two pieces
- **Tag for next step**: Set `carouselReady` and `infographicReady` flags appropriately

## Step 4: Write Output Files

Create directory: `micro-content/<source-slug>/`

Each micro-content piece is an MDX file:

**Filename convention:** `<NN>-<type>-<short-descriptor>.mdx`

Example: `01-hook-forget-half.mdx`, `02-thread-three-archetypes.mdx`

**Each file includes full frontmatter:**
```yaml
---
title: "[Descriptive title for the micro-content]"
type: "hook" | "thread" | "carousel" | "hot-take" | "faq" | "listicle" | "contrast"
platform: ["linkedin", "x", "instagram"]
sourceBlog:
  title: "[Source blog title]"
  slug: "[Source slug]"
  series: "[Series name]"
  part: [Part number]
  url: "[Relative path to source MDX]"
date: "[Today's date]"
author: "The Maximo Guys"
tags: [inherited from source blog]
status: "draft"
carouselReady: false
infographicReady: false
---

[Content here — formatted for the target platform(s)]
```

## Step 5: Generate Content Calendar Summary

Create `micro-content/<source-slug>/_index.mdx` with:

```yaml
---
title: "Micro-Content Index: [Source Blog Title]"
sourceBlog:
  title: "[Source blog title]"
  slug: "[Source slug]"
  url: "[Path]"
totalPieces: [count]
generatedDate: "[Today]"
---
```

Then a markdown table listing all generated pieces:

```markdown
| # | Type | Title | Platforms | Carousel | Infographic |
|---|------|-------|-----------|----------|-------------|
| 01 | hook | ... | LI, X, IG | No | No |
| 02 | thread | ... | X, LI | No | No |
| ... | ... | ... | ... | ... | ... |
```

## Step 6: Report

Output a summary:
- Total pieces generated by type
- Platform coverage (how many for LinkedIn, X, Instagram)
- Pieces ready for carousel conversion
- Pieces ready for infographic conversion
- Suggested posting schedule (1 piece/day = X days of content)
