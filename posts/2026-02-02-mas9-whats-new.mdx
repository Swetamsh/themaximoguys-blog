## Migration Guide

### From MAS 8 to MAS 9

**Prerequisites:**
- MAS 8.11+ (upgrade to latest MAS 8 first if needed)
- OpenShift 4.14+
- Database backup completed
- All integrations documented

**Step-by-step process:**

1. **Backup everything**
```bash
# Backup Maximo database
db2 backup database MAXDB to /backups/pre-mas9

# Export MAS configuration
oc get configmaps -n mas-core -o yaml > mas8-config-backup.yaml
oc get secrets -n mas-core -o yaml > mas8-secrets-backup.yaml
```

2. **Run upgrade preflight checks**
```bash
# Download MAS 9 upgrade checker
curl -o mas9-preflight.sh https://ibm.com/support/mas9-preflight
chmod +x mas9-preflight.sh

# Run validation
./mas9-preflight.sh --source-version 8.11 --target-version 9.0
```

3. **Execute upgrade**
```bash
# Apply MAS 9 operator
oc apply -f mas9-operator.yaml

# Trigger upgrade
oc patch Suite mas-instance -n mas-core \
  --type='merge' \
  -p '{"spec":{"version":"9.0.0"}}'

# Monitor upgrade progress
oc get suite mas-instance -n mas-core -w
```

4. **Post-upgrade validation**
- [ ] Login to Manage UI
- [ ] Run smoke tests
- [ ] Verify integrations
- [ ] Check mobile apps
- [ ] Test custom scripts

## Known Issues and Workarounds

### Issue 1: Slow initial login after upgrade

**Symptoms:** First login takes 60+ seconds

**Workaround:**
```bash
# Clear application cache
oc delete pods -l app=maximo-manage -n mas-core
```

### Issue 2: Custom icons missing

**Cause:** Icon library path changed

**Fix:**
```javascript
// Old path (MAS 8)
/maximo/ui/img/custom-icons/

// New path (MAS 9)
/maximo/ui/v9/assets/custom-icons/

// Update icon references in custom apps
```

### Issue 3: Mobile sync errors with large datasets

**Temporary limit:** Reduce sync dataset size to 10,000 records

**Permanent fix:** Expected in MAS 9.1 (Q2 2026)
