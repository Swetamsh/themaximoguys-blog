## What Is Nano Banana Pro

Nano Banana Pro is the community nickname for **Gemini 3 Pro Image**, Google DeepMind's state-of-the-art image generation and editing model released in November 2025. Key features:

- **"Thinking" process**: Reasons through your prompt before generating (plans composition, checks physics, verifies spelling)
- **Perfect text rendering**: Spells long sentences and complex logos accurately (99%+ accuracy under 400 words of text)
- **Search grounding**: Connects to Google Search for factually accurate diagrams
- **Up to 14 reference images**: Maintains strict brand or character consistency
- **Native 1K to 4K resolution** output
- **Conversational editing**: Edit existing outputs instead of re-rolling

## Golden Rules of Prompting

### 1. Natural Language Over Tags
```
BAD:  "Cool car, neon, city, night, 8k"
GOOD: "A cinematic wide shot of a futuristic sports car speeding through a rainy Tokyo street at night"
```

### 2. Edit, Don't Re-roll
The model excels at conversational edits. Request specific adjustments to existing outputs rather than regenerating from scratch.

### 3. Specificity Matters
- Define subjects precisely: "sophisticated elderly woman wearing vintage Chanel-style suit"
- Include material descriptions: "matte finish," "brushed steel," "soft velvet"
- Establish context about intended use

### 4. Provide Purpose
Include the "why" or "for whom" -- mentioning that an image is "for a Brazilian high-end gourmet cookbook" helps the model infer professional plating and lighting decisions automatically.

### 5. Talk to It Like a Human Artist
Use proper grammar and descriptive adjectives. Brief it as if you were briefing a human designer.

### 6. Treat Prompts as Technical Specifications
The model rewards **planning and intentionality**. Think of it as a "hyper-competent junior partner" -- provide architectural blueprints and strategic direction, not random keywords.

### 7. Text Limit: Keep Under 400 Words
For text-heavy projects, keep total text in your image prompt under 400 words. Going over can lead to hallucinations or garbled text.

## Prompt Structure Framework

### Basic 6-Element Structure
1. **Subject** (who/what)
2. **Composition** (camera angles, framing)
3. **Action** (what's happening)
4. **Setting/Location** (where)
5. **Style** (art type, aesthetic)
6. **Editing instructions** (for remixing)

### Advanced Elements
- Aspect ratios (16:9, 9:16, 1:1, 3:4, 5:2)
- Camera specs (lens type, focal length, lighting setup)
- Text rendering with typography details
- Color palette specifications (hex codes)
- Reference image roles

### Professional Prompt Layering
1. Subject description (clothing, positioning, attributes)
2. Composition rules (lighting, perspective, photography style)
3. Technical specifications (camera type, lens, resolution)
4. Typography & text requirements
5. Color palette & branding constraints

## Expert Prompt Engineering Techniques

### 1. JSON-Structured Prompts (for complex scenes)
```json
{
  "subject": "Young professional woman",
  "outfit": "Navy blazer, white shirt",
  "expression": "Confident smile",
  "lighting": "Three-point studio setup",
  "background": "Blurred modern office",
  "camera": "85mm f/1.4, eye-level",
  "mood": "Professional, approachable"
}
```

### 2. Logical Layout Anchors
For UX/UI design, brandbooks, posters, infographics, and presentations, provide detailed structural specifications for where elements should appear (headers, footers, sidebars) to achieve pixel-perfect layouts.

### 3. Typography Specification
```
"Use specific font families for different sections:
- Headlines: Bold sans-serif (like Inter or Roboto)
- Body text: Regular weight, high readability
- Accent text: Monospace for data/code elements
- Handwritten: For annotations and callouts"
```

### 4. Grid Generation for Variations
Creating 2x2 or 4x4 grids is cost-effective and allows distinct, contextually-aware variations within one generation. Each subimage maintains awareness of previous sections.

### 5. System Prompts for Consistency
Apply style constraints at the system level when generating multiple variations with consistent branding requirements.

### 6. Iterative Refinement Pattern
"Leave everything exactly the same, but [specific change]" -- preserves successful elements while making targeted adjustments.

### 7. Sketch-to-Finished Asset
Upload hand-drawn layout sketches for the model to follow structural guidance while applying professional rendering.

## What Works & What Doesn't

### WORKS WELL
- One-off premium header images with complex requirements
- Professional infographics requiring factual accuracy (leverage Google Search grounding)
- Product photography simulations with specific technical specifications
- Consistent brand visual variations using grid generation
- Text rendering in multiple languages
- Dense informational posters and educational materials
- Magazine-quality layouts with precise typography
- Code rendering for technical documentation
- Long-form prompts with detailed specifications

### DOESN'T WORK WELL
- Rapid iteration cycles (20 seconds to 1+ minute per generation)
- Highly stylized or surreal content (model pushes toward hyperrealism)
- Character consistency with unusual/non-standard designs
- Text exceeding 400 words in a single image
- Purely abstract or non-representational art

### PLATFORM-SPECIFIC TIPS
| Platform | Best Styles | Avoid |
|----------|------------|-------|
| LinkedIn | Corporate Memphis, Clean SaaS UI, Swiss Design | Overly casual, meme-style |
| Instagram | Glassmorphism, Bento Grid, Lifestyle | Text-heavy, small fonts |
| X/Twitter | Bold typography, High-contrast, Data viz | Cluttered, low-contrast |
| Blog Headers | Gradient backgrounds, Isometric, Editorial | Generic stock-photo look |
| Presentations | Flat design, Minimal, Dark mode UI | Overly decorative |
