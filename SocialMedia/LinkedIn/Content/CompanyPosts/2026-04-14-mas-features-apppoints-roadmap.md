## Post Copy

```
Two questions nobody answers honestly:

How much does this cost per user? How long does it actually take?

-> AppPoints are a flexible shared pool -- reallocate as you deploy new applications
-> Role-based allocation optimizes spending: Technician at 10 points vs Maintenance Manager at 30
-> The 13-month roadmap is realistic -- 6-month timelines usually fail because they skip data quality
-> Phase 0 (data quality) determines everything -- fix install dates, failure codes, and asset classification first
-> Success criteria per phase prevent premature advancement to the next phase

We have seen organizations buy thousands of AppPoints, allocate them on day one across every application, and wonder six months later why Health scores are meaningless and Predict models will not train.

The problem was never the technology. The problem was sequencing and data quality.
```
