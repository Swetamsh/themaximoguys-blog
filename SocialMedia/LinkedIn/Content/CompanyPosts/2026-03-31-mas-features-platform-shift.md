## Post Copy

```
MAS 9 is not a version upgrade. It is a re-architecture.

If someone on your team said "we just upgrade the EAR file and redeploy," this is for them.

Here is what changed under the hood:

-> Monolithic WebSphere EAR replaced by containerized microservices on Red Hat OpenShift
-> Java 17 is mandatory and will break most custom Java code
-> The database is sealed in managed deployments -- no direct SQL access
-> Infrastructure teams need Kubernetes, OpenShift, and container networking skills
-> Core business objects, Application Designer, and Automation Scripts remain unchanged

The business logic survived. The entire infrastructure did not.

Your team needs to understand this before they plan a single sprint.

#IBMMaximo #MAS9 #OpenShift #TheMaximoGuys #CloudNative
```
