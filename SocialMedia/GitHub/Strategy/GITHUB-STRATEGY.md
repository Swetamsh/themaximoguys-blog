# TheMaximoGuys GitHub Strategy

**Date:** March 2026
**Relevance:** 5/10
**Priority:** Tier 3 — Start Month 4

---

## Why GitHub

- No Maximo partner publishes open-source utilities or automation scripts
- Developers search GitHub directly for "Maximo automation script" solutions
- Establishes deep technical credibility (you cannot fake a useful repo)
- GitHub repos rank well in Google search

---

## Existing Maximo Repos to Know

- **maximo-wiki/maximo-automation-scripts** — Community automation scripts
- **ibm-maximo-dev/maximo-scripting** — Official IBM scripting repo
- **IBM/automation-maximo-app-suite** — MAS provisioning on OpenShift

---

| Repo | Source | Content |
|------|--------|---------|

---

## Rules

- Repos must contain **actually working code** (not marketing)
- Maintain regularly — stale repos damage credibility
- README links back to relevant blog posts
- Accept contributions / issues from the community
