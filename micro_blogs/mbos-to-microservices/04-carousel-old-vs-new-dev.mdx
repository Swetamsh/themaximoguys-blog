## Carousel: 7.6 Developer vs MAS 9 Developer

---

### Slide 1 — Title

**The Maximo App Developer: Then vs Now**

Everything you knew still matters. Some of how you work is changing.

*TheMaximoGuys.com*

---

### Slide 2

**Data Access**

THEN: Direct SQL queries against the database. Fast to write. Fragile at upgrade time.

NOW: GraphQL API. Ask for exactly what you need. Schema-defined. Upgrade-safe.

---

### Slide 3

**Business Logic**

THEN: Custom Java MBOs extending IBM's class hierarchy.

NOW: Automation scripts for most logic. Extensibility framework for the rest. External microservices for complex, independent workflows.

---

### Slide 4

**User Interface**

THEN: Dojo UI widgets. JSP modifications. Custom Java beans for complex UI logic.

NOW: React components. The UI is a client app that talks to APIs. Server-side UI logic is over.

---

### Slide 5

**Deployment**

THEN: Build WAR/EAR files. Deploy to WebSphere. Hope nothing breaks.

NOW: Publish APIs and events. Containers handle the runtime. Your code is more portable than it has ever been.

---

### Slide 6

**Observability**

THEN: Debugging with log files and prayer. Grep through server logs hoping to find the right thread.

NOW: Aggregated observability. Centralized logging, metrics, and tracing. You can actually see what your code is doing.

---

### Slide 7

**Version Control**

THEN: Version control? Export the XML. Save it somewhere. Hope the file names make sense in six months.

NOW: Git everything. Scripts, configurations, customizations — all in source control. Pull requests. Code review. Rollbacks.

---

### Slide 8

**Testing**

THEN: Testing meant clicking through the UI. "Works on my machine" was a valid sign-off.

NOW: Automated API tests. Test against the contract. Run in CI/CD before anything reaches production.

---

### Slide 9

**What Transfers 100%**

Your Maximo domain knowledge transfers completely.

You know work orders, assets, purchase orders, service requests. You know the data model. You know the business processes.

That knowledge is the hard part. The technology patterns are learnable.
