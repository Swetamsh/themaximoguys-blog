You do not need to learn everything before migration day. You need to learn the right things in the right order.

Here is the four-week learning path that builds the foundation you actually need as a MAS app developer:

---

**Week 1: GraphQL Fundamentals**

GraphQL is the primary data access layer in MAS 9. Learn the query syntax, understand schemas and types, practice traversing related objects in a single query, and run your first queries against the MAS GraphQL endpoint. Four hours of focused learning and it clicks.

---

**Week 2: REST API Patterns**

GraphQL is not the only API. REST is still how external systems trigger Maximo actions and integrations. Learn how to authenticate with the MAS APIs, how to structure CRUD operations, how to handle errors, and how to test APIs with tools like Postman or Insomnia before you write a line of code.

---

**Week 3: Container Basics**

You do not need to become a Kubernetes operator. You need to know enough to stop being afraid. Learn what a container image is, what a pod does, and how your automation scripts and extensions land inside the MAS runtime. Read logs from a container. Understand why "it works on my laptop" and "it fails in MAS" happen and how to close that gap.

---

**Week 4: Modern Dev Tooling**

Git for everything — no more XML exports. VS Code with the GraphQL extension for schema-aware development. A basic CI/CD pipeline concept so your scripts can be tested before they reach production. Linting and code review basics. One week of tooling investment that pays dividends every single day afterward.

---

Four weeks. Practical. One week at a time.

Your Maximo domain knowledge is already there. These four weeks give it a modern toolkit.
