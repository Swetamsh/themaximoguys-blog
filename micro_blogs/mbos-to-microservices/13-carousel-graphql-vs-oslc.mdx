## Carousel: OSLC vs GraphQL — The Maximo Data Access Evolution

---

### Slide 1 — Title

**How You Access Maximo Data Is Changing**

OSLC got us here. GraphQL takes us forward.

Here is what that shift actually means for your code.

*TheMaximoGuys.com*

---

### Slide 2 — The OSLC Problem: Over-fetching

**OSLC/REST: Returns everything.**

A standard OSLC work order request returns 200+ fields.

You needed six.

Your integration code strips out 194 fields on every single call. On every. Single. Call.

That is not a data access strategy. That is a workaround.

---

### Slide 3 — The GraphQL Answer: Precision

**GraphQL: Request exactly what you need. Nothing more.**

```graphql
query {
  workOrder(wonum: "WO1001") {
    wonum
    description
    status
    asset {
      assetnum
      description
    }
  }
}
```

Six fields. That is what you get. No stripping required.

---

### Slide 4 — The OSLC Problem: Multiple Round Trips

**OSLC: Multiple calls for related objects.**

Work order. Then asset. Then location. Then open service requests.

Four calls. Four round trips. Four chances to fail.

Latency adds up. Error handling multiplies.

---

### Slide 5 — The GraphQL Answer: One Query

**GraphQL: Traverse relationships in one query.**

Work order, asset, location, and open service requests — all returned in a single request.

One call. One round trip. One response to parse.

Your integration code gets simpler. Your network usage drops.

---

### Slide 6 — The OSLC Problem: Documentation Hunting

**OSLC: Documentation hunting.**

What fields does the work order object expose? Which are filterable? What are the valid status values?

Check the wiki. Check the old integration guide. Ask someone who has been here ten years.

Hope the documentation is current.

---

### Slide 7 — The GraphQL Answer: Self-Documenting

**GraphQL: Schema-defined, IDE auto-complete, self-documenting.**

The schema is the documentation. It is always current by definition.

Your IDE reads the schema. It auto-completes field names. It tells you the type of every field. It flags errors before you run a single query.

You stop hunting. You start building.
