## Carousel: The Maximo Mindset Shift — Old vs New

---

**Slide 1 — Hook**

Headline: The Maximo Mindset Shift

Subheadline: What changes when you move from 7.6 to MAS 9?

Everything but your domain knowledge.

Swipe to see the 8 shifts every Maximo professional must make.

Visual notes: Bold headline, dark background, TMG branding. Swipe indicator arrow.

---

**Slide 2 — Shift 1**

Label: CUSTOMIZATION

OLD WAY: Modify the product directly

NEW WAY: Extend via supported patterns

One line of context: Your extensions survive upgrades. IBM's code stays intact. Cleaner boundaries for everyone.

Visual notes: Two-column layout. Left side greyed (old), right side highlighted (new). Contrast color treatment.

---

**Slide 3 — Shift 2**

Label: ARCHITECTURE

OLD WAY: Server-side everything

NEW WAY: Client-side + API calls

One line of context: Logic that used to live in server-side scripts now calls well-defined APIs. Separation of concerns becomes real.

Visual notes: Server icon fading to API icon. Clean and minimal.

---

**Slide 4 — Shift 3**

Label: DATA ACCESS

OLD WAY: Direct SQL when needed

NEW WAY: GraphQL / REST APIs exclusively

One line of context: APIs are the contract. Direct database access is gone. This enables versioning, security, and client diversity.

Visual notes: Database icon with red X transitioning to API endpoint icon.

---

**Slide 5 — Shift 4**

Label: PROCESSING MODEL

OLD WAY: Batch-first, scheduled

NEW WAY: Event-driven, real-time

One line of context: Stop asking "when does this run?" Start asking "what event should trigger this?" Responses happen in real-time.

Visual notes: Clock icon (old) to lightning bolt icon (new). High contrast.

---

**Slide 6 — Shift 5**

Label: INTEGRATION

OLD WAY: Point-to-point integrations

NEW WAY: API gateway, event mesh

One line of context: Spaghetti integrations give way to decoupled, observable event streams and standardized API contracts.

Visual notes: Tangled lines (old) to clean hub-and-spoke diagram (new).

---

**Slide 7 — Shift 6**

Label: INFRASTRUCTURE

OLD WAY: Own the infrastructure

NEW WAY: Operate on shared platform

One line of context: IBM and Red Hat handle the platform. You focus on operating Maximo within it — higher leverage, less toil.

Visual notes: Server rack icon (old) to cloud platform icon (new).

---

**Slide 8 — Shift 7**

Label: TROUBLESHOOTING

OLD WAY: Manual monitoring, thread dumps, log files on a server

NEW WAY: Platform observability — Prometheus, Grafana, distributed traces

One line of context: Better visibility than before, once you know where to look. Logs, metrics, and traces are first-class citizens.

Visual notes: Stack of files (old) to clean observability dashboard graphic (new).

---

**Slide 9 — The Foundation**

Headline: Your domain knowledge transfers 100%.

Body: Work orders. Assets. Preventive maintenance. Workflow. Inventory. Purchasing.

All the same. All fully yours.

The technology layer shifts. Your expertise doesn't expire.

Visual notes: Centered text, strong typography, reassuring tone. Green or positive accent color.
