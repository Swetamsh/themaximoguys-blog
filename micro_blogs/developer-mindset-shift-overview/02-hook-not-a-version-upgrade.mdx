MAS 9 isn't a version upgrade. It's an architectural paradigm shift.

The database is different. The deployment model is different. The extension patterns are different. The operational model is different.

The only thing that carries over 1:1? Your domain knowledge. Work orders, assets, preventive maintenance, workflow, security groups -- the business logic is the same.

Everything else? New playbook.

If you're planning your MAS migration like it's a fix pack... you might want to rethink that.
